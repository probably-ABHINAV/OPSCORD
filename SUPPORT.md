# Support

Need help with OPSCORD? We've got you covered.

## Where to get help

Depending on the nature of your request, here are the best places to find support:

### 1. I have a question about how to use OPSCORD

If you are struggling with setting up the project locally, deploying, or understanding how a feature works:

- **Join our Discord:** Connect with the community and maintainers directly in the [OPSCORD Discord Server](https://discord.gg/opscord) (Placeholder link). This is the fastest way to get help.
- **GitHub Discussions:** Visit our [Discussions Tab](https://github.com/probably-ABHINAV/OPSCORD/discussions) and ask your question in the "Q&A" section.

### 2. I found a bug

If you are confident you have found a reproducible bug in the software:

- Please check the [Issue Tracker](https://github.com/probably-ABHINAV/OPSCORD/issues) first to ensure it hasn't already been reported.
- If it's a new issue, open a **Bug Report** using our issue template. Provide as much detail as possible (logs, screenshots, exact steps to reproduce).

### 3. I want to request a feature

If you have an idea to make OPSCORD better:

- Search existing issues and discussions.
- If it's a new idea, open a **Feature Request** using our issue template.

### 4. I found a security vulnerability

- **Do not open a public issue.**
- Please refer to our [SECURITY.md](./SECURITY.md) for instructions on how to safely disclose security issues.

---

_Note: OPSCORD is maintained by volunteers. Please be patient and respectful when requesting support._
